Web statique

Un site Web statique utilise le rendu côté serveur pour fournir des fichiers HTLM, CSS et Javscript prédéfinis à un naviguateur Web Contrairement aux sites dynamiques traditionnelles qui fonctionnent en affichant la page web au moment de la demmande. Les sites Web statiques contiennent un nombre fixe de pages et le format de page Web est fixe et fournit des informations au client. Le contenu d'une page Web est modifié pendant l'exécution de page sur le navigateur du client.

Lors de la consultation d'une page web statique, un serveur HTTP envoie le contenu du fichier dans lequel le contenu de la page est enregistré. Lors de la consultation d'une page web dynamique, un serveur HTTP transmet la requête à un logiciel qui génère le contenu de la page. Les logiciels générant des pages web dynamiques sont fréquemment écrits avec les langages PHP, JavaServer Pages (JSP) ou Active Server Pages (ASP). Une page web statique n'est programmée qu'avec les langages de programmation de base : HTML et (facultatif) CSS; à la différence des pages dynamiques. L'apparition de générateurs de sites statiques tels que Jekyll ou Hugo a rendu la génération de sites statiques plus simple en intégrant des langages de balisage léger, tels que le Markdown ou en permettant l'intégration de templates pour des parties récurrentes des pages comme l'en-tête ou le pied de page
